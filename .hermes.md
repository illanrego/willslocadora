# Locadora — Hermes project context

## Goal

Ship a local, genre-first 1990s rental-store web interface that uses native Stremio for the user's existing add-ons and playback.

## Product truth

Locadora is a discovery/experience layer, not a streaming service, torrent client, Stremio add-on, or player. The user selects a store year, browses genre shelves, and sends a selected title to native Stremio.

## Stack / layout

No application stack is selected or installed yet. Planning docs:

- `PRODUCT_SPEC.md` — product scope and phased delivery
- `ARCHITECTURE.md` — local UI/bridge/Stremio boundary
- `TASKS.md` — live task board
- `docs/integration-spike.md` — Phase 0 evidence and decisions

## Confirmed environment

Stremio is a running Flatpak app (`com.stremio.Stremio`), with its local engine at `127.0.0.1:11470`. Treat that engine as a playback/streaming component, not the catalogue source.

## Ship focus

1. Phase 0: verify exact item handoff and safe catalogue connection.
2. Phase 1: functional plain local shelves and counter-to-Stremio loop.
3. Phase 2: playtest and refine the rental loop.
4. Phase 3: add the 2.5D store presentation only after the loop works.

## Explicitly out of scope

- torrent/debrid/stream logic, playback, Stremio add-on work, player embedding
- private Stremio/Flatpak-file scraping and GUI automation
- hosted services, account systems, mobile app, full free-roam 3D

## Working rules

- Localhost only. Keep secrets and raw configured add-on URLs out of browser assets, console output, commits, and docs.
- Existing Stremio add-ons are the content/playback system. Catalogue-capable and stream-only add-ons have different roles.
- Do not install dependencies or select a framework before Phase 0 acceptance is reviewed with the user.
- Do not claim a title is playable; Stremio determines that after handoff.
- Prioritise a usable plain shelf navigation loop over visual spectacle.
- Do not git add, commit, or push; the user owns version control.
