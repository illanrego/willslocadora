# Locadora — Hermes project context

## Current direction

Will’s Locadora is a public, Brazil-first VHS rental discovery experience. Public browsing uses a static frontend plus a dedicated read-only Worker; durable rentals, reviews, and history are the next phase.

The authoritative brief is `MVP_PUBLIC_PRODUCT_AND_ARCHITECTURE.md`. `PRODUCT_SPEC.md` and `ARCHITECTURE.md` document the earlier local-first path and are not current deployment authority.

## Architecture

```text
Public static frontend (GitHub Pages)
  → locadora-api Cloudflare Worker
    → TMDB/public catalogue metadata

Future participation state
  → Supabase Auth + Postgres + RLS

Known-title action
  → visitor's registered Stremio handler
```

- `public/api-config.js` selects the local Node bridge only on local hosts and the public Worker elsewhere.
- `workers/locadora-api/` owns only public read endpoints and the Worker-side TMDB secret.
- The Worker is deployed separately with Wrangler; pushing Git does not deploy it.
- The Balcony is an intentionally local-state prototype until authenticated rentals exist.

## Non-negotiable boundaries

- No player, torrent/debrid, stream-resolution, Stremio add-on work, UI scraping, or private Stremio data.
- Catalogue and provider presence never guarantee playback or subscription access.
- No secrets in browser code, repository files, docs, logs, or fixtures.
- Use exact CORS origins; do not use wildcard CORS.
- Supabase anon configuration may be browser-visible when introduced; service-role credentials may not.

## Current delivery focus

1. User playtests the browse, title, shelf, and Balcony rental ritual.
2. Decide rental package size and due-date policy.
3. Implement Supabase Auth/profiles/RLS, then durable rentals/history, then reviews/community and deterministic recommendations.

## Working rules

- For Locadora, stage and commit only changes made for the current task; do not include unrelated work and never push unless asked.
- Run the relevant tests and `npm run build:pages` before committing application changes.
- A Worker code change also requires Wrangler deployment plus a real public endpoint check before it is called live.
